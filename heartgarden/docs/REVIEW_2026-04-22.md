## What is working well (don't break these)

- Apply path validates direct target spaces and merge target spaces against computed allowed scope before writes.
- Clarification validation/patch flow remains strict for required questions before apply.
- Import job processing emits structured phase/progress/LLM telemetry and supports cancellation checks.
- New scope radio is wired through UI -> userContext -> planning/apply path with schema-level validation.

## Recommended attack order

1. Restore merge acceptance UX wiring (highest user-facing correctness impact).
2. Add optimistic concurrency guard to merge writes (data integrity under multi-user edits).
3. Address performance hotspots (`nextZIndex` N+1 and spaces-search request/query behavior).
4. Harden trust boundary by deriving scope from server-owned source for apply.
5. Fix async race/event-trim concurrency edges.
6. Update docs (`/api/spaces/search`, `importScope`, review workflow changes).
