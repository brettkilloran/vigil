This review covers the two commits that finalized the "Global Links + Alt-graph" plan: a new `branes` table that lets links and a "mention graph" span multiple spaces, four new HTTP routes (`/api/branes/[id]/vocabulary`, `/api/graph/brane`, `/api/items/[id]/mentions`, `/api/mentions`), an Alt-hover discovery card on the canvas, and the removal of the old `[[wiki-link]]` typing assist. The biggest concern is one critical authorization ordering bug: the link edit and delete endpoints write to the database before checking whether the caller is allowed to touch the link's target space, so an unauthorized change can land before the user gets a `403`. A second cluster of high-severity issues is around cross-brane safety: items can be moved across branes (and folders re-parented across branes) without enforcing same-brane invariants that the link/mention layer assumes, and a single title edit can fan out into a brane-wide mention rebuild. There's also visible product drift — `[[wiki-links]]` are still produced by the lore importer but no longer typeable by users, and the demo brane ships without any mentions seeded, so Alt-discovery in the demo looks broken. Documentation for the new APIs and MCP tools has not caught up. Foundations are healthy: optimistic locking on item PATCH, source-side authorization on link edits, mention-table uniqueness/indexing, and cancellation/cache primitives in the Alt UI are all sound.

## What is working well (don't break these)

- Source-side player + GM authorization is correctly enforced before mutation in `/api/item-links` PATCH/DELETE; only the target-side check is in the wrong order.
- Item PATCH preserves optimistic-locking on `baseUpdatedAt`, so multi-tab edits do not silently overwrite each other.
- Cross-brane link creation is rejected at the validation layer (`validateLinkTargetsInBrane`), giving a clean place to extend the same guarantee to item moves.
- `entity_mentions` has strong table-level invariants — unique `(source, target, term, kind)`, CHECK that source ≠ target, and indexes on `brane_id` and `source_space_id` — which is a good base for scaling.
- Semantic neighbor scan in `entity-mentions.ts` is explicitly capped (`SEMANTIC_MAX_SOURCE_CHUNKS`, `SEMANTIC_PER_CHUNK_NEIGHBORS`, `SEMANTIC_MAX_TARGETS`), preventing unbounded vector fanout.
- `resolveOrCreateBraneByType` uses `ON CONFLICT DO NOTHING` on `brane_type`, which is the right pattern for first-writer-wins under concurrency.
- The Alt UI already includes `AbortController` cancellation and short TTL caches — good groundwork for the throttling fix.
- Boot-tier enforcement and GM-only gating are consistently applied across the four new mention/graph routes; only the per-item / per-brane scoping is missing.
