
- Analysis: The client requests `includeItemIds` only on first page, but coerces missing `itemIds` on later pages to an empty set. - Fix direction: Treat `itemIds === undefined` as `null` (no authoritative snapshot) and skip tombstone pruning for that page. Restrict full-subtree deletion logic to pages that explicitly carry a full ID snapshot. Add a regression test for page1/page2 sequencing.
