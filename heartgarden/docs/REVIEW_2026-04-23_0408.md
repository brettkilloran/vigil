## What is working well (don't break these)

- Optimistic locking exists in item patch flow and already enforces `baseUpdatedAt` for mutation-sensitive fields, providing a good concurrency foundation.
- Lore migration hooks are centralized in bridge/focus helpers, making it feasible to tighten canonical card handling in one layer.
- Player policy and type checks are already present in API flow; they need strengthening, not a full redesign.
- Focus/editor architecture already separates title/body/focus-surface concerns, which is a good structure for adding structured dirty tracking cleanly.

---

## Recommended attack order

1. Fix CRITICAL sync deletion behavior (`/changes` pagination + tombstoning) to stop potential local data loss.
2. Close HIGH-risk integrity/security gaps: HTML sanitization boundaries, player existing-type authorization, and protected merge structured-field conflicts.
3. Enforce canonical data model contracts: `entityType` allowlist and unified lore variant matrix across API/MCP/create/migration.
4. Repair editing-experience correctness: include roster in dirty/save gating; resolve location `ref` deprecation behavior explicitly.
5. Address performance hot paths (Set dedupe + revision/hash comparisons) to stabilize large-space editing responsiveness.
6. Finish cleanup of deprecated references and duplicated schema/transform logic to reduce future drift.
