## HIGH

### Lore import finalization is non-atomic and can violate cancellation terminal state

- **Severity:** HIGH
- **Confidence:** 0.94
- **Why it matters:** `processLoreImportJob` writes queue data and later writes terminal status in separate statements without a status guard/transaction. A concurrent cancel can be overwritten by the late `"ready"` write, or queue persistence can succeed while terminal status fails to advance. Either case creates user-visible state drift (cancelled jobs appearing ready, or persisted review queue with non-ready job state).

### Delta merge order can orphan moved items when their destination space row arrives in the same payload

- **Severity:** HIGH
- **Confidence:** 0.90
- **Why it matters:** Item patches are merged before space rows. If an item points to a destination space not yet present locally, `mergeRemoteItemPatches` updates the entity but cannot append it to `entityIds` (because that space key is missing). Later `mergeRemoteSpaceRowsIntoGraph` creates the space with `entityIds: []`, leaving the moved item effectively orphaned from space membership until another repair path runs.

## MEDIUM

### Space membership updates do repeated linear scans/filters under high churn

- **Severity:** MEDIUM
- **Confidence:** 0.89
- **Why it matters:** Two hot merge paths repeatedly scan or filter arrays (`entityIds`) for each changed/tombstoned item and for each conflict-apply operation. Under larger spaces/collaboration bursts this trends toward quadratic behavior and can increase UI merge latency.

```501:505:heartgarden/src/components/foundation/architectural-db-bridge.ts

    const home = entityHome.get(id);
    if (home && spacesRecord[home]) {
      spacesRecord[home].entityIds = spacesRecord[home].entityIds.filter((e) => e !== id);
    }
    entityHome.delete(id);
  };
```

```616:620:heartgarden/src/components/foundation/architectural-db-bridge.ts
  for (const sid of Object.keys(spacesRecord)) {
    const sp = spacesRecord[sid]!;
    if (!sp.entityIds.includes(item.id)) continue;
    spacesRecord[sid] = { ...sp, entityIds: sp.entityIds.filter((e) => e !== item.id) };
```

**Fix direction:** Maintain an id-to-space index (or map built once per merge) so removal/rehome is O(1) target lookup instead of scanning all spaces/arrays. Avoid repeated `filter` by using mutable-indexed removal or set-backed membership in hot paths, then serialize to arrays for persisted shape.

- **Severity:** MEDIUM
- **Confidence:** 0.84
- **Why it matters:** The fallback interval launches async fetches without an in-flight guard or abort controller. If requests run longer than the interval, polling can stack and increase redundant traffic/race windows for stale responses.

## Open questions/assumptions

- Assumption: delta `rawItems` can reference destination spaces not already present in local graph and concurrently included in `rawSpaceRows`; this is implied by merge API shape but should be confirmed against server ordering guarantees.
- Assumption: lore import queue persistence and status transition are intentionally allowed to be eventually consistent; if so, docs should explicitly define cancellation precedence and expected reconciliation.

## What is working well (don't break these)

- `useHeartgardenSpaceChangeSync` already has single-flight behavior and abort wiring for the main `/changes` poll loop, reducing overlap/race exposure in primary sync path.
- `mergeRemoteSpaceRowsIntoGraph` preserves existing `entityIds` for known spaces, which prevents accidental membership wipes during normal parent/name updates.
- Lore import flow records phase progress and error metadata (`errorCode`, `lastPhase`), which gives a good base for safer user-facing failure reporting.

## Recommended attack order

1. Fix lore import terminal-state race/non-atomic finalization first (user correctness and cancellation semantics).
2. Fix space/item merge ordering or add post-merge reconciliation to prevent orphaned moved items.
3. Add single-flight + abort to link-revision fallback polling to reduce redundant network/race behavior.
4. Optimize space membership hot paths (`mergeRemoteItemPatches`, `applyServerCanvasItemToGraph`) with indexed lookups.
5. Normalize error surfaces for lore import failures (safe user message + structured diagnostic code).
